Overview

MyOs is a minimal 64-bit operating system developed for learning and experimentation with modern OS development concepts. The project demonstrates core operating system principles including bootloading, memory management, filesystem operations, and shell command processing.

Language: Assembly & C
Architecture: x86_64
Bootloader: GRUB Multiboot
License: MIT

System Architecture

Core Components

  • GRUB Bootloader: Multiboot-compliant bootloader that initializes long mode (64-bit)
  • Kernel: Minimal kernel written in Assembly and C
  • VGA Terminal: 80x25 character display with color support
  • Shell: Interactive command-line interface with history
  • RAM Filesystem: In-memory hierarchical filesystem
  • Memory Manager: Simple heap allocator with tracking
  • PIT Timer: Programmable Interval Timer at 100 Hz
  • ATA Driver: Basic IDE disk interface

Bootloader & Initialization

MyOs uses GRUB as its bootloader, configured through a multiboot header. The boot process follows these steps:

  1. BIOS loads GRUB from the boot sector
  2. GRUB reads the multiboot header and loads the kernel
  3. CPU is switched to long mode (64-bit)
  4. Kernel initialization begins (GDT, IDT setup)
  5. VGA terminal is initialized
  6. PIT timer starts running
  7. RAM filesystem is created
  8. Shell prompt appears

Memory Management

Heap Allocator

MyOs implements a simple first-fit heap allocator with the following features:

  • Dynamic memory allocation (malloc/free)
  • Block merging to reduce fragmentation
  • Memory tracking and statistics
  • Heap verification for debugging
Memory Commands
mem      # Show heap statistics
testmem  # Allocate test blocks and verify heap

RAM Filesystem

The filesystem is entirely memory-based, providing fast access but requiring manual persistence:

Features

  • Hierarchical directory structure
  • File and directory support
  • Unix-like path navigation
  • Manual save/load to ATA disk

Shell Interface

The MyOs shell provides an interactive command-line interface with:

  • Command History: Use history to view past commands
  • Built-in Commands: Over 20 system and utility commands
  • File Operations: Full filesystem manipulation
  • System Info: Memory stats, uptime, disk information

See the Commands page for a complete reference.
